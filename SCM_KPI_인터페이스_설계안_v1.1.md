## 필드 검증 결과 요약

신규 필드 추가 없이 구현 가능한 이유 — 아래 필드들이 이미 존재함:

| KPI | 필요 기능 | 기존 필드 |
|-----|----------|-----------|
| 품절 KPI | 품절 시작일 | `판매상태 변경일` (MRP sync_parts) |
| 품절 KPI | 품절 유형 | `파츠품절 Status` — 재고부족/Capa이슈/품질이슈 구분 |
| 판매 KPI | 연도/월/주차 | `과업지시년` / `과업지시월` / `과업지시주` (SERPA task) |
| 판매 KPI | 매입 총금액 | `공급가액` 필드 직접 사용 |
| 원가율 | 마진율 | `마진율(전체)_DMK` / `C) 원가_최종` / `R) 판매가(최종)` |

| 90일 결제 | 담당자 확인 | `발주담당자` (task) |
| 25/26년 추이 | 연월 집계 | `지출월` (지출결의 — formula 필드) |

### 데이터 소스

- **테이블**: `sync_parts` (`tblYGM8wBxlZQOu1l`)

### 인터페이스 구성

#### 필터
```
판매상태 = "일시품절" OR 판매상태 = "의도적 품절"
```

#### 표시 필드 (순서대로)
1. 파츠명
2. 판매상태 (색상 강조)
3. 파츠품절 Status (품절 원인)
4. **판매상태 변경일** → 품절 시작일
5. 협력사_재입고예정일자
6. 협력사_재입고예정수량
7. 협력사_재고수량
8. partner (담당 협력사)
9. 재고담당자

#### 품절일수 계산 방법
에어테이블 인터페이스에서는 수식 필드 없이 `판매상태 변경일`을 표시하고,  
실제 **품절일수는 인터페이스 뷰에서 `판매상태 변경일` 기준 오름차순 정렬**로 파악.  
(날짜가 오래될수록 장기 품절)

#### 그룹화 옵션
- 옵션 A: `재고담당자` 기준 (담당자별 현황)
- 옵션 B: `파츠품절 Status` 기준 (원인별 분류)

#### 정렬
`판매상태 변경일` ASC (오래된 품절이 상단)

#### Summary 섹션 (인터페이스 상단)
- 총 품절 파츠 수: Count
- 재입고 예정일 없는 건수: Count (협력사_재입고예정일자 = empty)
- 협력사 재고 0인 건수: Count (협력사_재고수량 = 0)

#### 대체 협력사 여부 파악 방법
별도 필드 없이 `partner` 필드에 복수 협력사가 연결되어 있으면 대체 협력사 있음.  
인터페이스에서 `partner` 필드 확장 시 복수 표시 여부로 육안 확인 가능.  
→ 실제 단건 조회 시 MRP `sync_partner` 테이블 통해 확인.

## 3. 원가율 분석

### 데이터 소스
- **베이스**: SERPA (`appkRWtF2j99XgBTq`)  
- **테이블**: `task` (`tbllJFVBoFjmbQXLN`)

### 사용 필드 (기존 — 이미 원가/판매가 계산 완료)

| 필드명 | 필드 ID | 타입 | 역할 |
|--------|---------|------|------|
| `마진율(전체)_DMK` | `fldTyDk8vSb62Pias` | formula | **원가율 역수** (1 - 마진율 = 원가율) |
| `R) 판매가(최종)` | `fldfjxi8TxujE0sGo` | formula | 판매단가 |
| `C) 원가_최종` | `fldVRhMtN8vUpeNUl` | formula | 최종 원가 |
| `C1) 실제원가_DMK` | `fldZgdVd0f4hRL4bW` | formula | 실제 원가 (DMK 기준) |
| `C) 표준원가` | `fldL64uXwRjsgGWZr` | rollup | 표준 원가 |
| `공급가액` | `fldna9s9RVSQOJAz4` | formula | 실제 매입금액 |
| `마진율(재고제외)_DMK` | `fldIv8HaFAIYJ9ePK` | formula | 재고 제외 마진율 |
| `마진율검토_DMK` | `fldZfBudr8qEYNqVF` | singleSelect | 마진 검토 상태 |

> **주의**: `마진율(전체)_DMK`가 마진율이므로, **원가율 = 1 - 마진율**.  
> 예: 마진율 30% → 원가율 70%

### 인터페이스 구성

#### View 1: 원가율 위험 품목 (고원가율 우선)

**필터**: `R) 판매가(최종)` ≠ 빈값 (판매가 있는 건만)  
**정렬**: `마진율(전체)_DMK` ASC (낮은 마진 = 높은 원가율이 상단)  
**조건부 색상**:
- `마진율(전체)_DMK` < 0.3 → 🔴 위험 (원가율 > 70%)
- `마진율(전체)_DMK` < 0.5 → 🟡 주의 (원가율 50~70%)
- `마진율(전체)_DMK` ≥ 0.5 → 🟢 양호

| 표시 필드 | 역할 |
|----------|------|
| task_id | 품목 |
| 수주처 | 협력사 |
| 공급가액 | 매입금액 |
| R) 판매가(최종) | 판매가 |
| C) 원가_최종 | 원가 |
| 마진율(전체)_DMK | 마진율 |
| 발주지시수량 | 수량 |
| 과업지시년 / 과업지시월 | 기간 |

#### View 2: 고매출-저마진 제품 (많이 팔리는데 마진 낮은 제품)

**정렬 1**: `공급가액` DESC (매입금액 큰 순)  
**정렬 2**: `마진율(전체)_DMK` ASC (마진율 낮은 순)  
**필터**: 과업지시년 = 2026

→ 상단에 위치할수록 원가 절감 우선순위 높음

#### KPI 요약
- 전체 평균 마진율: `마진율(전체)_DMK` AVG
- 마진율 30% 미만 건수: COUNT (원가 위험)
- 총 매입금액 대비 표준원가 합계 비교

### 4-2. 하도급법 90일 결제대금 미지급 현황

#### 데이터 소스
- **베이스**: SERPA (`appkRWtF2j99XgBTq`)  
- **테이블**: `task` (`tbllJFVBoFjmbQXLN`)

#### 90일 계산 로직 (기존 필드만 사용)

| 기산점 | `과업지시일자` (`fldYEepX9s2G9knvL`, formula) |
|--------|-----------------------------------------------|
| 완료 판단 | `지출결의` 링크 필드(`fld1dkR348c9Hpfgm`) 비어있으면 **미지급** |
| 90일 초과 조건 | `과업지시일자` < TODAY() - 90일 = **기준일: 2026-02-28 이전 발주** |
| 제외 조건 | `매입결산진행` = "대상아님" 인 건 제외 |

> 매일 업데이트되는 기준일 = TODAY() - 90일  
> 2026-05-29 기준: **2026-02-28 이전 과업지시일자** + 지출결의 없는 건 = 90일 초과 미지급

#### 사용 필드 (기존)

| 필드명 | 필드 ID | 타입 | 역할 |
|--------|---------|------|------|
| `task_id` | `fldiEfna4G3t41J9L` | singleLineText | 발주 식별 |
| `수주처` | `fldVhPSugTYwKVNwa` | multipleRecordLinks | 협력사 |
| `과업지시일자` | `fldYEepX9s2G9knvL` | formula | **기산점** |
| `공급가액` | `fldna9s9RVSQOJAz4` | formula | 미지급 금액 |
| `지출결의` | `fld1dkR348c9Hpfgm` | multipleRecordLinks | **비어있으면 미지급** |
| `매입결산진행` | `fldl2IxTwmFHnnicZ` | multipleSelects | "대상아님" 제외용 |
| `발주담당자` | `fldGoCJp45B3JtULe` | multipleLookupValues | **담당자별 확인** |
| `표준적요명` | `fldXJriOCWpDGAKvY` | singleSelect | 품목 |

#### 인터페이스 구성

**View: 90일 초과 미지급 현황**

필터 1: `지출결의` = 비어있음 (isEmpty)  
필터 2: `과업지시일자` < 90일 전 날짜 (매일 수동 또는 automation으로 업데이트 필요)  
필터 3 (제외): `매입결산진행` ≠ "대상아님"  

정렬: `과업지시일자` ASC (가장 오래된 건 상단)  
그룹화: `발주담당자` (담당자별 미지급 현황)

| 표시 필드 | 역할 |
|----------|------|
| task_id | 발주 번호 |
| 수주처 | 협력사 |
| 표준적요명 | 품목 |
| 과업지시일자 | 기산일 (발주 생성일) |
| 공급가액 | 미지급 금액 |
| 발주담당자 | 담당자 |
| 매입결산진행 | 현재 결제 진행 상태 |

> ⚠️ **90일 날짜 필터의 한계**: 에어테이블 인터페이스 필터는 `TODAY() - 90`을 동적으로 지원하지 않음.  
> **해결방법**: `과업지시일자` 필터를 "is before" + "X days ago" = 90 으로 설정 (에어테이블 상대 날짜 필터 지원)

#### KPI 요약
- 90일 초과 건수: COUNT
- 90일 초과 미지급 총액: `공급가액` SUM
- 담당자별 미지급 건수: 그룹별 COUNT
